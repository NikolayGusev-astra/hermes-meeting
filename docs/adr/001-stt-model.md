# ADR-001: Speech-to-Text model selection

**Status:** accepted  
**Date:** 2026-07-22  
**Deciders:** Николай, Алексей, Евгений

## Context

Meeting-intelligence транскрибирует русскую речь с вкраплениями английских терминов (GitHub, repo, Codex, Hermes). Whisper `medium` даёт ошибки на английских словах: «гиг хаут» (GitHub), «репа» (repo), «кодекс» (Codex). `large-v3` точнее, но в 2-3× медленнее и требует больше RAM. Облачные API (OpenAI Whisper) дают лучшее качество, но противоречат принципу local-first.

## Decision

**Стратегия: progressive disclosure.**

1. **По умолчанию** — `small` (быстро, локально, английские ошибки ожидаемы)
2. **Опционально** — `medium` / `large-v3` через `--model large-v3` (точнее, медленнее)
3. **Опционально в будущем** — облачный fallback через `--allow-cloud` + OpenAI Whisper API
4. **Обвязка компенсирует** — garbage filter (детект Whisper-мусора), второй LLM-проход для нормализации терминов

Словарь замен («гиг хаут»→GitHub) не поддерживается — это бесконечный цикл. Вместо этого LLM сама нормализует термины по контексту во втором проходе.

## Consequences

- **+** Пользователь выбирает компромисс скорость/качество под свою машину
- **+** Облако остаётся опциональным, не обязательным
- **+** Обвязка (garbage filter + verify pass) работает одинаково для любой модели
- **−** `large-v3` на CPU медленный (22 мин записи → ~30-40 мин обработки)
- **−** Английские ошибки на `small` неизбежны без второго LLM-прохода
